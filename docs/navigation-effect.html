CheckBox.js


Navigation Effect


The checkbox navigational effect deals with adding controller buttons to a uniquely specified checkbox list. In order to uniquely specify a checkbox list, an id attribute must be added to the checkbox list container element. The sample html structure clearly reveals this.

HTML 

<div class="checkbox-wrapper">

  <!--parent checkbox (group)-->
  <div id="checklist-1" data-role="checkbox-list" data-bind="radio">

    <!--first checkbox section-->
    <div data-role="checkbox">
      <div checkbox data-color="@#615c988d|lime">
        <div marker></div>
        <div marker></div>
      </div>
      <input type="checkbox">
    </div> 
    
    <!--second checkbox section-->
    <div data-role="checkbox">
      <div checkbox data-color="@#615c988d|lime">
        <div marker></div>
        <div marker></div>    
      </div>
      <input type="checkbox">
    </div>
    
    <!--third checkbox section-->
    <div data-role="checkbox">
      <div checkbox data-color="@#615c988d|lime">
        <div marker></div>
        <div marker></div> 
      </div>
      <input type="checkbox">
    </div>
  
  </div>  

  <!-- add some control buttons -->  
  <div class="controls">
    <button id="b1"><i class="bi-chevron-left"></i></button>
    <button id="b2"><i class="bi-chevron-right"></i></button>
  </div>

</div>

In the sample structure above, the id of the checkbox list is specified as checkbox-1. Once we uniquely specify a checkbox list, we can easily fetch the in-built navigation controller methods available in the plugin through the checkList() method. The sample below reveals this

JS 
let checkbox = new CheckBox(true);

checkbox = checkbox.check();
controllers = checkbox.checkList('checklist-1');

/*fetch checklist's navigation controller methods*/
console.log(controllers);

The navigation controller methods available are the prev(), next(), switch() and stop() methods. These methods are discussed below

let checkbox = new CheckBox(true);

checkbox = checkbox.check({

    assign: {
        alter: (dataAnime) => {
            /* use a default attribute value */
            dataAnime.unshift('animated');
            return dataAnime.map( el => 'animate__' + el );        
        }
    }

});

/* bind controller buttons */
let prevBtn = document.querySelector('#b1');
let nextBtn = document.querySelector('#b2');

prevBtn.addEventListener('click', () => {
    checkbox.checkList('group-1').prev() 
})

nextBtn.addEventListener('click', () => {
    checkbox.checkList('group-1').next() 
})
SCREEN

From the sample above, by adding a unique id attribute to the checkbox-list container div item, we were able to easily bind controller buttons to the checkbox group by first selecting the unique field through its id attribute using the checkList() method. Once this field is successfully selected, an object is returned that contains the prev() and next() methods which are respectively used to select the previous or next checkbox item. This feature makes it easier to control the checkbox flow. An easier way to bind control buttons is through the use of data-controls attribute whose values are unique ids that points respectively to a left and right navigation buttons. STOP

The controller methods prev() and next() can be easily iterated continuously. These methods supports an interval argument in milliseconds that determines how fast or slow the iteration should be. Once any of the methods is iterated, then the iteration can be terminated using the stop() method. A sample of this is shown below:
SCREEN

From the sample above, the forward and backwards controller buttons are iterated when click until the stop (middle) button is clicked. This is a good way to set controller navigation buttons for the checkbox list. However, when it comes to navigation controllers, we can easily set up navigation controllers by adding the data-nav attribute to the checkbox list. A sampleof this is further discussed in Navigation Effects.
Another method is the switch() method which pratically selects a checkbox within a list of checkboxes using its postional index starting from the range of one(1) to the maximum number of available checkboxes. For example, in the sample below, the second checkbox is easily selected in the list of three checkbox items.
JS 
/* sample: selecting input field*/
let numField = document.querySelector('#numField');

numField.addEventListener('change', () => {
  let value = this.value;
  check1.checkList('group-1').switch(value) 
})
SCREEN

Note that in the sample above, the data-bind attribute is a set as radio . Under a different option, the response will be different.
