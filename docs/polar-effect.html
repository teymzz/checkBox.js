Polar effect


The polar toggle effect is a bit similar to radio button effect except that this relatioship is only between the two markers of a single custom checkbox. Each of the markers will be the only active buttons to toggle each other. This means that when a marker is active, it cannot toggle itself off because it will require the action of the second marker to switch. In order to achieve this effect, both markers will be visible. Hence, data-flip needs to be set as false and also, the data-control attribute will be set as ":markers".

HTML CSS JS 
<div class="in-flex mid" style="padding: 10px; gap:.5em">
    <div data-role="checkbox" class="in-flex wrap">
        <div checkbox polar data-flip="false" data-control=":markers" data-anime="fadeIn|fadeIn" data-size="20" data-color="@red|green" data-assign=":markers">
            <div marker></div>         
            <div marker></div>
        </div>
        <input type="checkbox" value="polar">
    </div>
</div>

[polar] [marker] {
    border-radius: 100vh;
    background-color: #180b2b;
}

[polar] [marker]:nth-child(1) {
    background-color: rgb(255, 0, 120, 0.3);
}

[polar] [marker]:nth-child(2) {
    background-color: rgb(31, 162, 168, 0.3);
}

[polar][checked="unchecked"] [marker]:nth-child(1) {
    background-color: rgb(255, 0, 120);
}

[polar][checked="checked"] [marker]:nth-child(2) {
    background-color: rgb(31, 162, 168);
}

                          Polar markers' effect.

In the sample above, to uncheck the checkbox, the first marker has to be clicked. Similarly, to check the checkbox, the second marker has to be clicked. The brighter of the two colors helps us to know the current state of the checkbox. The polar effect means that each marker is the direct opposite of its sibling. A marker cannot inversely modify its active state unless its relative opposite marker is clicked. The main attributes for achieving a polar effect are the data-flip and the data-control attributes as shown in the sample above.
