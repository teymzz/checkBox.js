CheckBox.js


Toggle Attributes - :marker1


When using markers, we can specify that a custom checkbox should toggle the value from an attribute into another attribute on the the first custom checkbox marker when the checkbox is checked. A sample of this is shown below:

HTML JS (GLOBAL) 
<div checkbox class="foo" data-switch="bar">
  <div marker style="background-color: red;"></div>
  <div marker style="background-color: green;"></div>
</div>
<input type="checkbox">

<script>
new CheckBox({
  assign: {
      target: '[checkers]',
      attr: ['data-switch', 'class', ':marker1']
  }    
});
</script>

From the sample html structure, the bar value will be toggled into the class attribute of the first marker when the checkbox is checked due to the global configuration. Alternatively, we can set this modifier option locally by using the data-assign html attribute. A sample of this is shown below :

HTML CSS JS 
<div checkers class="foo" data-switch="bar" data-assign=":marker1|data-switch|class">
  <div marker style="background-color: red;"></div>
  <div marker style="background-color: green;"></div>
</div>
<input type="checkbox">

[checkers] {
    width: fit-content;
    border-radius: 100vh;    
    border: dotted 2.5px #514e68; 
    padding: 4px;    
}

[checkers] > [marker] {
    width: 60px;
    height: 2px;    
    background-color: transparent;
    transition: width .5s ease-in-out, 
                background-color .5s ease-in-out;
}

[checkers] > [marker]:first-child {
    width: 30px;
    background-color: rgba(149, 15, 15, 0.66);
}

[checkers] > .bar[marker] {
    width: 120px;                            
    background-color: red;
}

[checkers] > [marker] {
    border-radius: 100vh;
    background-color: green;
    margin: 4px;
}

<script>
new CheckBox({
  target: '[checkers]',
});
</script>
SCREEN

From the sample above, the bar value will be added into the class attribute of the first marker when the checkbox is checked and it will be removed when the checkbox is unchecked.
