Eager Loading (Toggle)


Eager loading is necessary in situations where we need to preload a toggle event. We have already seen how to initialize an event using the init option or the data-init attribute. Also, we have learnt about the toggle option or data-func attribute. We can easily combine these features to create a smart eager loading for the checkbox toggling. The sample below shows how to achieve this :

HTML JS 
<div checkbox="">
  <div marker=""></div> 
  <div marker=""></div>
</div>
<input type="checkbox" value="foo"> <br> 
<div class="response"></div>

<script>
new CheckBox({
    init: function(checkbox) {
        
        let value = checkbox.value;
        let textField = document.querySelector(".response");

        if( checkbox.checked ) {
            textField.innerHTML = `checkbox ${value} is: checked`
        } else {
            textField.innerHTML = `checkbox ${value} is: unchecked`
        }

    },
    toggle: function(checkbox) {
        
        checkbox.init(checkbox);

    }
});
</script>    

                 

                  

                    
 
                      

                      

                    

                     
                  

                  

                     

From the javascript above, we can see that the checkbox argument supplied into the toggle() callback option has access to the init() callback. This makes it easier to run the init() within the toggle() function rather than redefining it. This means that init() function will first run when the page loads and it will continue to run when the checkbox is toggled. This is also why the message in the display screen is showing earlier before the checkbox is being toggled.
