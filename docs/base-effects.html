Base Effects


The base effects are effects that have close relationship with a parent custom checkbox element. The checkbox "base" effect is one in which multiple checkboxes are binded to a root checkbox which is always the first checkbox item in a list of checkbox items. The base effect is mostly used in cases where we need to imitate the effect of one checkbox over another. This effect means that the activity of one checkbox is directly related to the activity of another checkbox item. In order to do this, the data-bind value will be set as "base". A few samples are shown below:

Base Effect HTML JS 
<div data-role="checkbox-list" data-bind="base">

  <!--first checkbox section-->
  <div data-role="checkbox" class="in-flex wrap">
    <div checkers data-anime="fadeIn" data-assign=":marker2" data-color="@red|lime" class="in-flex">
    
      <div marker class="b-red"></div>
     
      <div marker class="b-lime"></div>
      
    </div>
    <input type="checkbox" value="N1">
  </div> 
  
  <!--second checkbox section-->

Base Effect
In the sample above, the first checkbox is the parent checkbox that controls the rest of the two checkboxes which are the children boxes. Whenever the parent checkbox is toggled on or off, the other two children checkboxes will respond similarly. If both of the children checkboxes are not toggled on, then the parent checkbox is never toggled on. A real usage of this feature is shown in the sample below:

The sample above provide a more real scenario where the checkbox effect can be applied. Although, the entire custom checkbox is grouped under the "base" effect. We can easily detach any children checkbox from their parent checkbox controller by specifying the data-bind attribute of that individual child as free similarly to the sample below:
Detach Checkbox HTML 
<div data-role="checkbox-list" data-bind="base" style="flex-direction: column;">

<!--first checkbox section-->
<div class="in-flex mid-v">
    <div data-role="checkbox" class="in-flex wrap rad-r">
        <div checker data-anime="fadeIn" data-assign=":marker2" data-label="Mark all children|Unmark all children" data-color="@white|lime" class="in-flex rad-r">
        
        <div marker class="b-currentColor"></div>
        
        <div marker class="b-currentColor"></div>
        
        </div>
        <input type="checkbox" value="radio1">
        
    </div>

In the sample html structure above, the last custom checkbox is set to a free mode by applying data-bind="free" attribute on the native checkbox that corresponds to the displayed custom box. This means that the relationship between the first (parent) checkbox and the last checkbox is detached. In other words, the activity of the parent checkbox will not affect the last checkbox.
