CheckBox.js


Toggling Attributes (Advanced)


Previously, we have seen how to toggle a source attribute's value into a destination attribute. However, in the previous discussion, we only applied a single source attribute. The checkbox plugin provides an easy way to define a maximum of two values for the source attribute which are separated by pipe. In this situation, the first value defines a value that is expected to be assigned when the checkbox is checked while the second value will be applied when the checkbox is unchecked. To understand this better, consider the html structure below:

HTML CSS 
<div checkbox source="foo|bar" data-assign="checked|source|class">
  <div marker=""></div>
</div>
<input type="checkbox">

[checkers] {
    width: fit-content;
    border-radius: 5px;    
    border: solid; 
    padding: 4px;    
}

[checkers] > [marker] {
    width: 20px;
    height: 20px;    
    background-color: transparent;
    transition: width .5s ease-in-out, 
                background-color .5s ease-in-out;
}

[checkers] > [marker] {
    width: 50px;
    background-color: green;
}

.bar[checkers] > [marker] {
    width: 30px;
    background-color: red;
}

From the sample html above, we have a source attribute that contains two destination values which are "foo" and "bar". When the checkbox is checked, the foo value will be assigned into a class attribute but if the checkbox is unchecked, the bar value will be assigned instead. The data-assign attribute uses the modifier option “checked” to specify that the destination value should be assigned on the custom checkbox itself. The type of modifier option applied will determine how the toggling effect is resolved. We also applied some simple css for width animation. The response will be similar to that which is shown below


This toggling effect above is usually applied in cases where we need to apply a smooth toggling effect. For example, the major use case of this feature can been seen when applying css animations especially from external libraries. Let's consider more sample structures with different modifiers below :

modifier (unchecked) 
<div checkbox source="foo|bar" data-assign="unchecked|source|class">  
    <div marker=""></div>
</div>
<input type="checkbox">

In the sample above, the "unchecked" modifier option will switch the behaviour of the source attribute such that "foo" will be toggled into the checkbox's class attribute when the checkbox is checked while "bar" will be toggled in when the checkbox is unchecked

modifier (:marker1) 
<div checkbox source="foo|bar" data-assign=":marker1|source|class">  
    <div marker style="background-color: red;"></div>
    <div marker style="background-color: lime;"></div>
</div>
<input type="checkbox">

In the sample above, the ":marker1" modifier option will switch the behaviour of the source attribute such that "foo" will be toggled into the first marker's class attribute when the checkbox is unchecked while the second value "bar" will be ignored. This means that only the first marker is affected because no value toggling activity will be performed on the second marker.

modifier (:marker2) 
<div checkbox source="foo|bar" data-assign=":marker2|source|class">   
    <div marker style="background-color: red;"></div>
    <div marker style="background-color: lime;"></div>
</div>
<input type="checkbox">

In the sample above, the :marker2 modifier option will switch the behaviour of the source attribute such that "bar" will be toggled into the second marker's class attribute when the checkbox is checked while the first value "foo" will be ignored. This means that only the second marker is affected because no value toggling activity will be performed on the first marker.

modifier (:markers) 
<div checkbox source="foo|bar" data-assign=":markers|source|class">   
    <div marker style="background-color: red;"></div>
    <div marker style="background-color: lime;"></div>
</div>
<input type="checkbox">

In the sample above, the ":markers" modifier option will switch the behaviour of the source attribute such that "foo" will be toggled into the first marker's class attribute when the checkbox is unchecked while "bar" will be toggled into the second marker's class attribute when the checkbox is checked.
