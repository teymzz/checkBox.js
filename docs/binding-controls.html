CheckBox.js


Binding Controls


The checkbox control button functionality is an integrated feature that is specifically designed to navigate through a checkbox list. In order to use this feature, a checkbox list must have a unique id attribute whose id value is supplied into the checkList() method. Once the checkList() method successfully pulls the object of the specified unique id, it returns controller methods which includes the next(), prev() and switch() that can be used to navigate through the checkbox list. The example below reveals the function of these methods:

HTML CSS JS 
  
<!--parent checkbox (group)-->
<div class="checkbox-wrapper">
  <div id="group-1" data-role="checkbox-list" data-bind="radio">

    <!--first checkbox section-->
    <div data-role="checkbox">
      <div checkbox data-anime="bounceIn" data-assign=":marker2" data-color="@#615c988d|lime">
      
        <div marker></div>
        <div marker></div>
      
      </div>
      <input type="checkbox" value="C1">
    </div> 
    
    <!--second checkbox section-->
    <div class="rotate" data-role="checkbox">
      <div checkbox data-anime="bounceIn" data-assign=":marker2" data-color="@#615c988d|lime">
      
        <div marker></div>
        <div marker></div>
      
      </div>
      <input type="checkbox" checked="true" value="C2">
    </div>
    
    <!--third checkbox section-->
    <div data-role="checkbox">
      <div checkbox data-anime="bounceIn" data-assign=":marker2" data-color="@#615c988d|lime">
      
        <div marker></div>
        <div marker></div>
      
      </div>
      <input type="checkbox" value="C3">
    </div>
  
  </div>  

  <!-- add some control buttons -->  
  <div class="controls">
    <button id="b1"><i class="bi-chevron-left"></i></button>
    <button id="b2"><i class="bi-chevron-right"></i></button>
  </div>
</div>

[data-role="checkbox-list"] {
  display: inline-flex;
  gap: .5em;  
}
  
[data-role="checkbox-list"] [marker] {
  width: 20px;
  height: 20px;  
}
  
[checkbox] {
  display: inline-flex;
  border: dotted 2px;  
  border-radius: 100vh;  
  padding: 4px;  
}
  
[checkbox] [marker] {
  width: 20px;
  height: 20px; 
  border-radius: inherit;    
  background-color: currentColor;    
}

.rotate[checkbox][checked="checked"] {
    animation: rotate 5s infinite linear;;
}

@keyframes rotate {

    0% {
      transform: rotate(0deg);
    }
    
    100% {
      transform: rotate(360deg);
    }
    
}

let checkbox = new CheckBox(false);

checkbox = checkbox.check({

    assign: {
        alter: (dataAnime) => {
            /* use a default attribute value */
            dataAnime.unshift('animated');
            return dataAnime.map( el => 'animate__' + el );        
        }
    }

});

/* bind controller buttons */
let prevBtn = document.querySelector('#b1');
let nextBtn = document.querySelector('#b2');

prevBtn.addEventListener('click', () => {
    checkbox.checkList('group-1').prev() 
})

nextBtn.addEventListener('click', () => {
    checkbox.checkList('group-1').next() 
})
SCREEN

From the sample above, by adding a unique id attribute to the checkbox-list container div item, we were able to easily bind controller buttons to the checkbox group by first selecting the unique field through its id attribute using the checkList() method. Once this field is successfully selected, an object is returned that contains the prev() and next() methods which are respectively used to select the previous or next checkbox item. This feature makes it easier to control the checkbox flow. An easier way to bind control buttons is through the use of data-controls attribute whose values are unique ids that points respectively to a left and right navigation buttons. A sample of this is shown below:

The controller methods prev() and next() can be easily iterated continuously. These methods supports an interval argument in milliseconds that determines how fast or slow the iteration should be. Once any of the methods is iterated, then the iteration can be terminated using the stop() method. A sample of this is shown below:
SCREEN

From the sample above, the forward and backwards controller buttons are iterated when click until the stop (middle) button is clicked. This is a good way to set controller navigation buttons for the checkbox list. However, when it comes to navigation controllers, we can easily set up navigation controllers by adding the data-nav attribute to the checkbox list. A sample of this is further discussed in Navigation Effects.
Another method is the switch() method which pratically selects a checkbox within a list of checkboxes using its postional index starting from the range of one(1) to the maximum number of available checkboxes. For example, in the sample below, the second checkbox is easily selected in the list of three checkbox items.

JS 
/* sample: selecting input field*/
let numField = document.querySelector('#numField');

numField.addEventListener('change', () => {
  let value = this.value;
  check1.checkList('group-1').switch(value) 
})
SCREEN

Note that in the sample above, the data-bind attribute is a set as radio . Under a different option, the response will be different. Also, when switching the state of checkboxes using the switch() method as shown above, a second argument can help to specify the type of activity expected to be fulfilled through the options "on", "off", "both".
