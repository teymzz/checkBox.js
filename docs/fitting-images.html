CheckBox.js


Fitting images


The benefit of markers is that they can be used for loading images. By default, background images usually require additional css in order to be able to align well in view. The fit global configuration option provides an additional css to help fit images into checkbox. This feature depends on the custom box selector to determine the type of css style generated. For example, assuming that we have an custom checkbox structure below:

Custom CheckBox 
<div checkbox="">
  <div marker="" style="background-image: url(checkBox/src/images/check-1-blue.png);"></div> 
  <div marker="" style="background-image: url(checkBox/src/images/check-1-blue-fill.png);"></div>
</div>
<input type="checkbox"> <br>

In the sample above, we can see that the markers are used to anchor background images. Assuming that the custom checkbox selector is [checkbox]. We can try to fit the image into the markers using the script format below

Global Fit Image Option 
<script>
new CheckBox({
    fit: true // fit background images
});
</script>

Alternatively, we can use the local data-fit attribute to fit images just as shown below

Local Fit Image Attribute 
<div checkbox="" data-fit="true">
  <div marker="" style="background-image: url(src/images/check-1-blue.png);"></div> 
  <div marker="" style="background-image: url(src/images/check-1-blue-fill.png);"></div>
</div>
<input type="checkbox"> <br>

Based on the selector, this will generate a css that resembles the format below:

Fit css (auto-generated) 
[checkbox] [marker] {
    background-size: cover;
    background-position: center top;
    background-repeat: no-repeat;
}

The sample response will resemble the html view below

HTML View (Toggle on checkbox)

In situations where a much better styling is required, it is advised to define a custom css to handle this design.
