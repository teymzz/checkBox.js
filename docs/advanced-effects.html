CheckBox.js


Advanced Effects


The checkbox plugin supports extended relationships within a grouped list of custom checkboxes that are closely related. This group of two or more custom checkboxes is referred to as “checkbox list” and they are binded under the parent group identified with the html attribute of data-role and its value set as "checkbox-list". Once we place the custom checkboxes inside this parent group, we can then proceed to define the type of toggle effect we need to achieve. The sample structure for setting up advance effects is shown below:

Relative checkboxes 
<!--parent checkbox (grand group)-->
<div data-role="checkbox-list">

  <!--first checkbox section-->
  <div data-role="checkbox" class="in-flex wrap rad-r">
    <div checker data-anime="fadeIn" data-assign=":marker2" data-color="@red|lime" class="in-flex rad-r">
    
      <div marker class="b-red"></div>
      <div marker class="b-lime"></div>
      
    </div>
    <input type="checkbox" value="1">
  </div> 
  
  <!--second checkbox section-->
  <div data-role="checkbox" class="in-flex wrap rad-r">
    <div checker data-anime="fadeIn" data-assign=":marker2" data-color="@red|lime" class="in-flex rad-r">
    
      <div marker class="b-red"></div>
      <div marker class="b-lime"></div>
      
    </div>
    <input type="checkbox" value="2">
  </div>
  
  <!--third checkbox section-->
  <div data-role="checkbox" class="in-flex wrap rad-r">
    <div checker data-anime="fadeIn" data-assign=":marker2" data-color="@red|lime" class="in-flex rad-r">
    
      <div marker class="b-red"></div>
      <div marker class="b-lime"></div>
      
    </div>
    <input type="checkbox" value="3">
  </div>
  
</div>

From the sample above, the data-role attribute is used to specify both the parent and the children of a checkbox list. While the parent is specified as "checkbox-list", its direct children are specified as "checkbox" which helps to sectionalize each custom checkbox. Once this html structure is defined, the plugin will assume that all the children checkboxes belong to the parent checkbox-list and this will force the plugin to set up a relationship amongst them based on the type of relationship effect applied. There are basically six types of effects available which are the "radio", "base", "slide", "rating", "polar" and "navigation" effects. These effects are discussed on the next page.
