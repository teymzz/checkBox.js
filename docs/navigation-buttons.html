CheckBox.js


Navigation Buttons


Previously, we have seen how to navigate a checklist using the navigation controller methods provided by the plugin. We can however make this process much easier if all we need are controller buttons. The checkbox plugin makes it easier to add navigation buttons using the data-nav attribute. This attribute is used to define the unique ids of buttons that must handle the left and right navigations respectively. Consider the html structure below:

HTML 
<div id="checklist-1" data-role="checkbox-list" data-nav="left|right">

    <-- add custom checkbox sections here -->

</div> 

<button id="left">&#10094;</button>
<button id="right">&#10095;</button>

In the sample above, the data-nav attribute defines two unique ids "left" and "right" separated by pipe. Each of these id will respectively select the element whose id attribute's value is relative. Hence the two buttons above will be selected. The left button will automatically navigate the custom checkbox to the left while the right checkbox will navigate the custom checkbox to the right when it is clicked upon. This makes it easier to navigate the checkbox without adding any complex javascript code. We can similarly add the interval argument for both the left and right buttons by supplying a third option "click" along with the specified interval. This is shown below

HTML 
<div id="checklist-1" data-role="checkbox-list" data-nav="prev|next|click-500">

    <-- add custom checkbox sections here -->

</div> 

<button id="prev">PREV</button>
<button id="next">NEXT</button>

The third option supplied above into data-nav attributes is one of three supported animation effects that will be discussed later. This option defined above will set an iteration interval of 500ms when any of the navigation buttons are selected. If we need to add a stop button, the stop button selector must be added before the animation effect. The sample structure of this is shown below

HTML 
<div id="checklist-1" data-role="checkbox-list" data-nav="prev|next|stop|click-500">

    <-- add custom checkbox sections here -->

</div> 

<button id="prev">PREV</button>
<button id="next">NEXT</button>
<button id="stop">STOP</button>

From the example provided below, we can see the effect of the button's relationship on the checklist element.

HTML JS 
  
  <div id="checklist-1" data-role="checkbox-list" data-bind="radio" data-nav="prev|next|stop|click-500">
  
    <!--first checkbox section-->
    <div data-role="checkbox">
      <div checkbox data-color="@#615c988d|lime">
        <div marker></div>
        <div marker></div>
      </div>
      <input type="checkbox">
    </div> 
    
    <!--second checkbox section-->
    <div data-role="checkbox">
      <div checkbox data-color="@#615c988d|lime">
        <div marker></div>
        <div marker></div>
      </div>
      <input type="checkbox">
    </div>
    
    <!--third checkbox section-->
    <div data-role="checkbox">
      <div checkbox data-color="@#615c988d|lime">
        <div marker></div>
        <div marker></div>
      </div>
      <input type="checkbox">
    </div>
  
  </div>  
  
  <!-- add some control buttons -->  
  <div class="controls">
    <button id="prev">&#10094;</button>
    <button id="stop">&#9724;</button>
    <button id="next">&#10095;</button>
  </div>  
  

  let checkbox = new CheckBox(false);
  
  checkbox = checkbox.check({ flip: true });
  
  /* bind controller buttons & input field */
  let prevBtn = document.querySelector('#prev');
  let nextBtn = document.querySelector('#next');
  let stopBtn = document.querySelector('#stop');
  let switchField = document.querySelector('#switch');
  
  /* fetch the checklist field & add event to controllers*/
  let checkList1 = checkbox.checkList('checklist-1')
  
  prevBtn.addEventListener('click', () => {
      checkList1.prev(1000) 
  })
  
  nextBtn.addEventListener('click', () => {
      checkList1.next(1000) 
  })
  
  stopBtn.addEventListener('click', () => {
      checkList1.stop() 
  })
  
  switchField.addEventListener('change', () => {
      checkList1.stop() /*stop any previous animation*/ 
      checkList1.switch(this.value) 
  })
