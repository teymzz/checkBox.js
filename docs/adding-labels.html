Adding Labels


Labels are texts that immediately follows the input checkbox fields which are identified by the data-access="label" attribute. In order to add labels, the custom checkbox is assigned with an attribute of data-label which contains the text label itself that is either single or separated with pipe. When the text is separated by pipe, the left side of the text will be for when the checkbox is unchecked while the right side of the text will stand for the displayed text when it is checked. A sample format and sample view are displayed below.

Sample Label HTML CSS 
 <div class="checkbox-wrapper">
    <div checker data-label="Show|Hide" data-flip="true">
        <div marker style="background-color: transparent;"></div>
        <div marker style="background-color: lime;"></div>
    </div>
    <input type="checkbox" value="foo">
    <label data-access="label">Show</label>
 </div>  

[checkers] {
    width: fit-content;
    border: solid 2px #514e68; 
    border-radius: 100vh;
    display: inline-flex; 
    padding: 4px;    
}

[checkers][checked="checked"] {
    animation: rotate 5s infinite linear;;
}

[checkers] > [marker] {
    border-radius: 100vh;
}

@keyframes rotate {

    0% {
      transform: rotate(0deg);
    }
    
    100% {
      transform: rotate(360deg);
    }
    
}

From the sample structure above, we can see that the checkbox element has an attribute of data-label which contains two values "Show" and "Hide". Also, the label text field where the text will be added is identified with the attribute of data-access and it immediately follows the native input checkbox element. Whenever the checkbox is unchecked, the "Show" text will be added to the specified label field and the first marker will be displayed. However, if the checkbox is checked, the second marker will be displayed while the returned text will be "Hide".

Alternatively, we can shift the label outside the direct parent of the custom checkbox element. However, in this situation, the direct parent must have an attribute data-role with the value of "checkbox". A sample of this is shown below:

Alternative Labelling HTML 
<div class="checkers box">
  <div>
    <div class="checkbox-wrapper" data-role="checkbox">
      <div data-label="Show|Hide" class="in-flex">
        <div marker style="background-color: red;"></div>
        <div marker style="background-color: lime;"></div>
      </div>
      <input type="checkbox" value="N">
    </div>
    <div data-access="label">Show</div>
  </div>
</div>

From the sample structure above, the specified label text field will still be accessible by the custom checkbox because the direct parent of the custom checkbox has its data-role attribute set as "checkbox" while the data-access label text field comes immediately after it.
