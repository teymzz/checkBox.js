CheckBox.js


Callback Methods and Arguments


The checkbox callback method which are init() and toggle() along with callback attributes which are data-init and data-func are designed to have access to custom checkbox properties through the first supplied argument. The following properties are the internally generated properties of a custom checkbox target.

checkbox properties description
checked This method returns true only when a custom checkbox is checked
value This method returns the current equivalent value of a native checkbox
listId This method is used to determine the positional index of a custom checkbox within a checklist
disable This method returns true only when a native checkbox is disabled.
native This method returns the equivalent native checkbox element of a selected custom target.
custom This method returns the equivalent custom checkbox element of a selected custom target.
marker This method returns an array of a relatively selected checkbox marker found within a custom checkbox target.
parent This method returns the equivalent custom checkbox wrapper identified with data-role="checkbox". The value of this will be undefined if no checkbox wrapper is detected.
checkList For any custom checkbox that exist within a checklist identified with the data-role="checkbox-list" attribute, this method returns the equivalent parent checklist element. The value of this will be undefined if no parent checklist wrapper element is detected.
fit This property returns true if the icon fitting option of a checkbox on its markers is set as true
flip This property returns true if the flipping toggle effect option of a checkbox on its markers is set as true
checkedBoxes For any custom checkbox child that lies within a checklist parent, this property returns total number of checked custom checkboxes within the parent checklist element of that particular custom checkbox child.
color This returns the currently defined color for a custom checkbox target.
label This returns the currently defined label for a custom checkbox target.
init() This method is only accessible within the toggle() callback configuration option. It is used to access the init() method within the same supplied configuration object.

The example provided below is an illustration of how these properties can be accessed.

HTML JS 
<div checkbox="" data-func="myFunction">
  <div marker=""></div> 
  <div marker=""></div>
</div>
<input type="checkbox" value="foo"> <br> 
<div class="response"></div>

<script>
new CheckBox({
    init: function(props) {
        
        console.log(props); /*checkbox properties*/

    },
    toggle: function(props) {
        
        console.log(props); /*checkbox properties*/

    }
});

/* callback function defined by "data-func" attribute */
function myFunction(props) {
        
    console.log(props); /*checkbox properties*/

});
</script>
