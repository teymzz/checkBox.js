CheckBox.js


Markers (double)


The double markers' structure is similar to the single markers except that two markers are added into the custom checkbox container instead of a single marker. The first marker usually stands for when the native checkbox is unchecked while the second marker stand for when it is checked.

HTML CSS JS 
<div checkbox>
  <div marker=""></div> <!-- @unchecked -->
  <div marker=""></div> <!-- @checked -->
</div>
<input type="checkbox">

[checkbox] {
    width: fit-content;
    display: flex;
    border-radius: 5px;
    border: solid;
    padding: 4px;
}

[checkbox] > [marker] {
    width: 20px;
    height: 20px;
}

[checkbox][checked="checked"] > [marker]:nth-child(1) {
    opacity: .1;
}

[checkbox]:not([checked="checked"]) > [marker]:nth-child(2) {
    opacity: .1;
}

[checkbox] > [marker]:nth-child(1)  {
    background-color: white;
}

[checkbox] > [marker]:nth-child(2) {
    background-color: lime;
}

let checkbox = new CheckBox(true);

checkbox.check({

  marker: '[btn]' //custom marker selector

});
SCREEN

From the structure above, based on the css, the first marker will be displayed when the checkbox is unchecked while the second marker will be displayed when the checkbox is checked.
