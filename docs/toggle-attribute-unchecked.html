CheckBox.js


Toggle Attributes : unchecked


Similarly to the checked modifier, we can specify that a custom checkbox should assign the value from an attribute into another attribute when the custom checkbox is unchecked and should remove it when it is checked. Consider the sample structure shown below,

HTML JS (Global) CSS 
<div checkbox class="foo" data-switch="bar">
<div marker style="background-color: #594fc8;"></div>
</div>
<input type="checkbox">

<script>
new CheckBox({
        assign: {
            attr: ['data-switch', 'class', 'checked']
        }    
    });
</script>

From the sample above, due to the plugin's global configuration, in the html section, the value "bar" from the data-switch attribute will be toggled into the class attribute when the custom checkbox is unchecked. This was done using the plugin's assign configuration option. Alternatively, we can do this locally using the data-assign attribute as shown below :

HTML (Local) JS CSS 
<div checkers class="foo" data-switch="bar" data-assign="unchecked|data-switch|class">
  <div marker style="background-color: #594fc8;"></div>
</div>
<input type="checkbox">

<script>
new CheckBox();
</script>    

[checkers] {
    width: fit-content;
    border-radius: 5px;    
    border: solid; 
    padding: 4px;    
}

[checkers] > [marker] {
    width: 20px;
    height: 20px;    
    background-color: transparent;
    transition: width .5s ease-in-out, 
                background-color .5s ease-in-out;
}

[checkers] > [marker] {
    width: 50px;
    background-color: green;
}

/* set css for when bar is added */

.bar[checkers] > [marker] {
    width: 30px;
    background-color: red;
}

From the sample above, the bar value from the html attribute data-assign will be assigned into the class attribute when the checkbox is unchecked and it will be removed when the checkbox is checked.
