CheckBox.js


Toggle Attributes : both


This modifier is used when we need to add a value from a source attribute to a destination attribute both in cases when a checkbox is checked or unchecked. In this case, when a toggle event occurs, the specified value is first removed from the destination attribute before it is re-assigned back into it. Consider the sample structure shown below,

HTML JS (Global) 
<div checkbox class="foo" data-switch="bar">
<div marker style="background-color: #594fc8;"></div>
</div>
<input type="checkbox">

<script>
new CheckBox({
        assign: {
            attr: ['data-switch', 'class', 'both']
        }    
    });
</script>

From the sample above, due to the plugin's global configuration, in the html section, the value "bar" from the data-switch attribute will first be removed from the class attribute it exists before it is re-toggled back into it. This activity will happen in both cases when the checkbox is checked or unchecked. Alternatively, we can do this locally using the data-assign attribute as shown below :

HTML (Local) JS CSS 
<div checkers class="foo" data-switch="bar" data-assign="both|data-switch|class">
  <div marker style="background-color: #594fc8;"></div>
</div>
<input type="checkbox">

<script>
new CheckBox();
</script>    

[checkers] {
    width: fit-content;
    border-radius: 5px;    
    border: solid; 
    padding: 4px;    
}

[checkers] > [marker] {
    width: 20px;
    height: 20px;    
    background-color: transparent;
}

[checkers][checked="checked"] > [marker] {
    background-color: lime;
}

[checkers][checked="unchecked"] > [marker] {
    background-color: red;
}

/* set css for when bar is added */

.bar[checkers] > [marker] {
    width: 50px;
    transition: width .2s ease-in-out;
}

From the sample screen above, we can see the effect of the removal and re-assiging of the bar value on the class attribute in both cases when the checkbox is either checked or unchecked.
