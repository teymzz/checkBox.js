CheckBox.js


Toggle Attributes : checked


We can specify that when a custom checkbox is checked, the value from a source attribute should be assigned into another attribute and this value will also be removed when the checkbox is unchecked. In order to do this, let's create a sample html structure below.

HTML JS (Global) 
<div checkbox class="foo" data-switch="bar">
  <div marker style="background-color: #594fc8;"></div>
</div>
<input type="checkbox">

<script>
new CheckBox({
        assign: {
            attr: ['data-switch', 'class', 'checked']
        }    
    });
</script>

From the sample above, due to the plugin's global configuration, in the html section, the value "bar" from the data-switch attribute will be toggled into the class attribute when the custom checkbox is checked. This was done using the plugin's assign configuration option. Alternatively, we can do this locally using the data-assign attribute as shown below :

Global: assign value on checked 
<script>
new CheckBox({
assign: {
    target: '[checkers]',
    attr: ['data-switch', 'class', 'checked']
}    
});
</script>
HTML (Local) JS CSS 
<div checkers class="foo" data-switch="bar" data-assign="checked|data-switch|class">
  <div marker style="background-color: #594fc8;"></div>
</div>
<input type="checkbox">

<script>
new CheckBox();
</script>    

[checkers] {
    width: fit-content;
    border-radius: 5px;    
    border: solid; 
    padding: 4px;    
}

[checkers] > [marker] {
    width: 30px;
    height: 20px;    
    background-color: #5d5c6288;
    transition: width .5s ease-in-out, 
                background-color .5s ease-in-out;
}

/* set css for when bar is added */

.bar[checkers] > [marker] {
    width: 30px;
    background-color: lime;
}

From the sample above, the bar value from the html attribute data-assign will be assigned into the class attribute when the checkbox is checked and it will be removed when the checkbox is unchecked.
