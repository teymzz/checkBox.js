CheckBox.js


Binding Controls


The checkbox control button functionality is an integrated feature that is specifically designed to navigate through a list of custom checkboxes within a checkbox group list. In order to use this feature, a checkbox group list must have a unique id attribute after which the methods next(), prev() and to() becomes available to select or navigate through the checkbox group list. The example below reveals the function of these methods:

HTML CSS JS 
  
<!--parent checkbox (group)-->
<div class="checkbox-wrapper">
  <div id="group-1" data-role="checkbox-list" data-bind="radio">

    <!--first checkbox section-->
    <div data-role="checkbox">
      <div checkbox data-anime="bounceIn" data-assign=":marker2" data-color="@#615c988d|lime">
      
        <div marker></div>
        <div marker></div>
      
      </div>
      <input type="checkbox" value="C1">
    </div> 
    
    <!--second checkbox section-->
    <div class="rotate" data-role="checkbox">
      <div checkbox data-anime="bounceIn" data-assign=":marker2" data-color="@#615c988d|lime">
      
        <div marker></div>
        <div marker></div>
      
      </div>
      <input type="checkbox" checked="true" value="C2">
    </div>
    
    <!--third checkbox section-->
    <div data-role="checkbox">
      <div checkbox data-anime="bounceIn" data-assign=":marker2" data-color="@#615c988d|lime">
      
        <div marker></div>
        <div marker></div>
      
      </div>
      <input type="checkbox" value="C3">
    </div>
  
  </div>  

  <!-- add some control buttons -->  
  <div class="controls">
    <button id="b1"><i class="bi-chevron-left"></i></button>
    <button id="b2"><i class="bi-chevron-right"></i></button>
  </div>
</div>

[data-role="checkbox-list"] {
  display: inline-flex;
  gap: .5em;  
}
  
[data-role="checkbox-list"] [marker] {
  width: 20px;
  height: 20px;  
}
  
[checkbox] {
  display: inline-flex;
  border: dotted 2px;  
  border-radius: 100vh;  
  padding: 4px;  
}
  
[checkbox] [marker] {
  width: 20px;
  height: 20px; 
  border-radius: inherit;    
  background-color: currentColor;    
}

.rotate[checkbox][checked="checked"] {
    animation: rotate 5s infinite linear;;
}

@keyframes rotate {

    0% {
      transform: rotate(0deg);
    }
    
    100% {
      transform: rotate(360deg);
    }
    
}

let checkbox = new CheckBox(false);

let check1 = checkbox.check({

    assign: {
        alter: (dataAnime) => {
        
            /* use a default attribute value */
            dataAnime.unshift('animated');
            return dataAnime.map( el => 'animate__' + el );
            
        }
    }

});

/* bind controller buttons */
let prevBtn = document.querySelector('#b1');
let nextBtn = document.querySelector('#b2');

prevBtn.addEventListener('click', () => {
    check1.customList('group-1').prev() 
})

nextBtn.addEventListener('click', () => {
    check1.customList('group-1').next() 
})
SCREEN

In the sample above, notice that the attribute "checked" was set as "true" on the second native checkbox. This will notify the checkbox to start by checking the checkbox. A "false" value will always keep it unchecked when the web page is loaded. This helps to prevent any cache storage that may occur most especially in desktop browsers.
